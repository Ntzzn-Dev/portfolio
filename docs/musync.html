Musync

Autor: Jhonatan | Nathan

GitHub: https://github.com/Ntzzn-Dev/Musync

Data: 05/07/2025

Descrição

Um aplicativo mobile com capacidade de organizar músicas locais, reproduzi-las, baixar e transferir as músicas entre plataformas.



Demonstração


Tela Principal

Lista

Nessa tela vemos logo de cara nossa playlist principal.

A playlist principal por padrão é a "Todas" [1] que exibe todos os arquivos de áudio encontradas pelo aplicativo.

Mas também há a possibilidade de alterar para uma pasta em específico, ou uma playlist criada por você.

Pesquisa

Nessa tela também é possivel pesquisar por músicas que estão na playlist em exibição

Apertando na lupa, um campo de texto [2] aparece, possibilitando a pesquisa de músicas presentes na playlist principal.

Tela Principal

Player

Na tela principal e na tela da playlist, um player simples é exibido na parte inferior, com os controles principais da faixa em execução:

  • Slider de tempo
  • Shuffle (modo aleatório)
  • Prev (faixa anterior)
  • Play/Pause
  • Next (faixa posterior)
  • Repeat (modo de repetição)

Ao ser clicado em um espaço vazio, o player esconde os controles [3], exibindo apenas o titulo e o autor da faixa.

Seleção de telas

Download

O botão download [4] leva para a área de download.

Configurações

O botão Configurações [5] leva para uma tela com configurações simples do aplicativo.

Super Control

O botão SuperControl [6] leva para uma tela que funciona como um player com botões maiores, ideal para usar como interface de um rádio por exemplo, sem distração, apenas botões e comandos.

Connect to Desktop

O botão Connect to Desktop [7] abre a camera para que possa escanear o QRCode gerado na versão de desktop

Tela Principal

Checkpoint

O botão superior com o icone ▶, serve como um backup, pois o app guarda a ultima playlist e música ouvida antes do app fechar.

Após clicar no botão, ou definir uma música como checkpoint, o status muda de 'ultima musica ouvida' para 'checkpoint marcado' [14], alterando também o icone para algo parecido com ⊕.

Nesse estado o comportamento é praticamente o mesmo, ao clicar no botão, começa a tocar a música marcada com o checkpoint.

Tela de Configurações

A tela de configurações possui:

  • Playlist padrão que a tela de downloads carrega. (link de uma playlist do youtube)
  • Diretórios onde as músicas são baixadas pelo downloader.
  • Botões de reset para ups & desups
  • Modo de energia

Na tela de download, a playlist padrão é carregada permitindo a seleção de uma ou mais mídias para download via vídeos do YouTube. Enquanto o diretório para downloads é usado como pasta final para faixas baixadas.

Modos de energia

Modo economico

O aplicativo deixa de rodar em segundo plano, portanto caso fique tempo sem estar no foco (tocando musica ou usando o app) o android fecha ele para economizar memória e bateria.

Modo balanceado

O aplicativo roda em segundo plano por 1 hora, após isso o android pode fecha-lo para economizar memória e bateria.

Modo Performance

O aplicativo fica em segundo plano até que o próprio usuário feche-o, causando assim uma maior consistência de dados, evitando que a musica deixe de ser carregada (precisando abrir novamente), porém possui um gasto maior de bateria.

Tela Downloads

Ao abrir a tela de downloads, a sua playlist do youtube, definido na Tela de configurações, é carregada com os 200 primeiros itens, separados por pedaços de 10.

Nessa tela, após carregar a playlist, torna-se possível selecionar as músicas que deseja fazer o download.

Caso queira usar uma playlist diferente da padrão para selecionar músicas, basta colar o link no campo.
Caso queira baixar uma faixa em específico, que não está em nenhuma playlist, basta colar o link do video no campo.

Após ter selecionado as faixas, deve clicar no botão inferior, que inicia o download.

A ordem utilizada é crescente, ou seja da faixa mais antiga à mais atual (de baixo pra cima).

Após isso inicia o ciclo de download, permitindo que saia dessa tela.

Ciclo de Downloads

Após ter iniciado os downloads, não se é mais necessário, ficar na mesma tela de downloads, pois as faixas são baixadas de maneira assincrona.

Na tela inicial é possivel ver um botão que indica os downloads em processo [8].

Clicando nesse botão você pode ver o progresso atual de quantas faixas ja foram baixadas.

Após concluir todos os downloads que foram selecionados, o botão muda de icone [9] indicando que ja foi finalizado.

Clicando nesse botão você recebe as músicas baixadas, sem a necessidade de reiniciar o app.

Playlists

Nessa aba, logo ao lado da lista principal, ficam as setlists, que nada mais são que um amontoado de diretórios.

A ordem de setlists sempre será a mesma:

  • Playlists
  • Pastas
  • Artistas

As playlists são criadas pelo próprio usuário, dando titulo e subtitulo

As pastas são divisões automáticas que o app faz, sendo que "/todas" é uma pseudo-pasta que armazena todas as faixas escaneadas pelo aplicativo

Abaixo vem todos os artistas de todas as músicas, também são pastas separadas automáticamente pelo app.

E assim como na aba principal é possivel buscar por músicas, nessa aba vc pesquisa por setlists.

Notificação

Assim que uma música começa a tocar, um player é assinado no android, ficando na barra de notificações.

Esse mini player possui alguns comandos simples, que permite manipular o comportamento sem necessidade de abrir o app:

  • Slider de tempo
  • Shuffle (modo aleatório)
  • Prev (faixa anterior)
  • Play/Pause
  • Next (faixa posterior)
  • Up

Caso o aplicativo esteja usando o modo economico, é possivel que o android mate o processo, mas não retire a notificação. Então mesmo que ainda esteja com nome e foto da ultima faixa na notificação, é necessário reiniciar o app para voltar a funcionar.

Shuffle (modo aleatório)

Modo aleatório padrão com um modo a mais:

Off

Não causa alterações na lista, segue de maneira ordenada

On

Modo padrão, embaralha as músicas aleatóriamente, mas mantém o histórico de músicas ja ouvidas, portanto o shuffle é apenas pra frente (seja dando next manualmente ou automaticamente quando uma faixa termina)

Optional

Parecido com o modo padrão, embaralha as músicas aleatóriamente mantendo o histórico, porém ele segue a ordem da lista padrão, e apenas pula para uma música aleatória quando o botão next é apertado. Isso foi feito para que sequencias de músicas pudessem ser reproduzidas em ordem, mas quando chegasse em uma que não estivesse agradando, um unico botão te joga para um lugar aleatório da lista.

Utilização

  1. Clone este repositório: 
    git clone https://github.com/Ntzzn-Dev/Musync.git
  2. Para Android: 
    cd musync_and
flutter pub get
flutter run
  3. Para Desktop: 
    cd musync_dkt
flutter pub get
flutter run
